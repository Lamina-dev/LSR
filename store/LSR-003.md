# LSR 003 - Lamina C 扩展与插件规范

## 基本信息

- LSR 编号 003
- 标题 Lamina C 扩展与插件规范
- 作者 Ziyang-Bai
- 状态 草案
- 类型 标准规范
- 创建日期 02-04-2026
- 归属项目 编译器、虚拟机、包管理器

## 摘要

本规范定义 Lamina 与原生 C/C++ 扩展的集成方式，包括包结构、描述文件、接口映射语法和内存安全约束。目标是让扩展在性能、类型安全、生命周期管理上保持一致行为。

## 技术规范

### 1. 目录结构

扩展包必须采用以下结构：

- `[root]/lampm.json`：包元数据与构建配置
- `[root]/lib.lm`：Lamina 侧接口声明文件
- `[root]/src/`：C/C++ 源码目录
- `[root]/include/`：私有头文件目录（可选）

### 2. `lampm.json` 字段

`lampm.json` 必须至少包含下列字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| `name` | String | 全局唯一包名 |
| `type` | String | 固定为 `"extension"` |
| `language` | String | C/C++ 标准版本，如 `"C11"`、`"C++17"` |
| `build.cflags` | Array | 编译参数，如 `[-O3, -march=native]` |
| `build.libs` | Array | 链接库列表，如 `[m]` |
| `interface` | String | 接口文件路径，默认 `"lib.lm"` |
| `entry` | String | 动态库入口函数名，默认 `"lsr_init"` |

示例：

```json
{
  "name": "example",
  "version": "1.0.0",
  "type": "extension",
  "language": "C11",
  "build": {
    "cflags": ["-O3", "-ffast-math"],
    "libs": ["m"]
  },
  "interface": "lib.lm",
  "entry": "lsr_init"
}
```

### 3. 接口映射语法

在 `lib.lm` 中使用 `module` 声明扩展命名空间，函数签名遵循 LSR-000：

- 静态强类型
- 参数与返回类型必须显式
- 不使用 `:` 类型标注

```lamina
module example {
  # 语法：func name(param Type, ...) -> ReturnType = "c_symbol"
  func det(m matrix) -> num = "c_ext_det"
  func eig(m matrix) -> matrix = "c_ext_eig"
}
```

### 3.1 `table` 类型接口约束

当接口参数或返回值使用 `table<K, V>` 时，必须满足：

- `K` 必须可哈希
- 默认允许键类型：`num`、`complex`、`text`、`bool`
- 默认禁止键类型：`vector`、`matrix`、`set`、`table`，以及未声明哈希语义的自定义类型

不满足约束时，编译器必须在接口绑定阶段报错，并停止生成桥接代码。

```lamina
module ext_table {
  func score_of(t table<text, num>, name text) -> num = "c_ext_score_of"
  # 非法：vector 不能作为 table 键
  # func bad(t table<vector, num>) -> num = "c_ext_bad"
}
```

### 3.2 复合对象内存安全与所有权协议

为保持 LSR-000 的值语义与 COW 语义，`table`、`matrix`、`vector` 传入扩展时必须声明访问模式：

- `@readonly`：只读视图，禁止原地写
- `@move`：所有权转移，扩展接管生命周期

默认模式是 `@readonly`。

#### 3.2.1 引用计数约束

- `@readonly` 绑定必须增加持有计数，释放时归还
- `@move` 绑定必须使 Lamina 侧引用失效，避免双重释放
- 扩展不得持有未声明所有权的裸指针

#### 3.2.2 写入前克隆要求

在 `@readonly` 模式下，如需写入，必须调用运行时接口：

`clone_if_shared(obj)`

语义：

- 独占对象：直接返回可写对象
- 共享对象：先复制再返回可写对象

未调用该接口直接写共享对象，运行时必须拦截为 `ExtensionMutationViolation`。

#### 3.2.3 接口示例

```lamina
module ext_safe {
  func frobenius(a matrix @readonly) -> num = "c_ext_frob"
  func consume(t table<text, num> @move) -> bool = "c_ext_consume"
}
```

说明：`@readonly` / `@move` 为接口注解。未声明时默认 `@readonly`。

### 4. 错误与诊断

推荐至少支持以下错误码：

- `InterfaceBindError`：接口映射失败
- `TableKeyTypeError`：`table` 键类型不合法
- `OwnershipViolation`：所有权协议违规
- `ExtensionMutationViolation`：未按规则写共享对象

### 5. 与其他 LSR 的关系

- LSR-000：核心类型系统、值语义、COW 语义
- LSR-004：标准库定义，扩展可作为补充实现
- LSR-008：量纲剥离规则（扩展无量纲入参场景）
